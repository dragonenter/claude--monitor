# 炉石传说酒馆战棋 AI 自动打牌插件 — PRD

## 项目概述

AI 自动打牌插件，针对炉石传说酒馆战棋模式，能自动读取游戏状态、AI 决策、自动执行操作，实现全自动完成一局酒馆战棋对局。

**目标平台：** Windows 10/11 64位
**游戏语言：** 仅支持简体中文客户端
**AI 水平目标：** 中等水平（天梯 6000-7000 分），前四率 ≥ 50%

---

## 功能列表

### 游戏状态读取（双方案）

| 编号 | 功能 | 优先级 | 说明 |
|------|------|--------|------|
| F-01 | 日志文件监听引擎 | P0 | tail-follow 模式监听 output_log.txt，插件式解析架构 |
| F-02 | 日志解析 — 商店随从 | P0 | 从日志提取当前酒馆可购买随从列表（名称、攻防、种族、费用） |
| F-03 | 日志解析 — 手牌 | P0 | 从日志提取手牌中随从/法术信息 |
| F-04 | 日志解析 — 己方战场 | P0 | 从日志提取场上随从（名称、攻防、位置、关键词、种族） |
| F-05 | 日志解析 — 资源信息 | P0 | 从日志提取金币数、酒馆等级、生命值、护甲值、升级费用 |
| F-06 | 日志解析 — 英雄与技能 | P0 | 从日志提取英雄身份、技能状态（可用/已用/费用） |
| F-07 | 日志解析 — 回合与阶段 | P0 | 从日志提取回合数、识别游戏阶段（招募/战斗/选英雄/结束） |
| F-08 | 日志解析 — 对手信息 | P1 | 从日志提取存活对手列表、血量、上次对战对手 |
| F-09 | 截图识别（备用） | P1 | 日志无法覆盖的字段，每回合截一帧补充识别 |
| F-10 | GameState 统一数据结构 | P0 | 日志+截图输出统一 GameState，AI 仅依赖此结构 |
| F-11 | 三连进度追踪 | P1 | 追踪场上+手牌中同名随从数量，标记即将三连 |

### AI 决策引擎

|------|------|--------|------|

| F-13 | 出售随从决策 | P0 | 决定出售手牌/场上随从以腾空间或回收金币 |
| F-14 | 刷新酒馆决策 | P0 | 决定是否花1金币刷新 |
| F-15 | 升级酒馆决策 | P0 | 权衡血量、经济、阵容强度决定升本时机 |
| F-16 | 随从站位决策 | P0 | 考虑嘲讽、亡语、攻击顺序排列随从 |
| F-17 | 英雄技能决策 | P0 | 判断何时用技能及选择目标 |
| F-18 | 阵容方向规划 | P1 | 根据种族池和已有随从规划中长期阵容方向 |
| F-19 | 三连追求策略 | P1 | 判断是否值得为凑三连持有/购买特定随从 |
| F-20 | 英雄选择 | P1 | 游戏开始时根据英雄评级表自动选择 |

|------|------|--------|------|

## 验收标准

### 状态读取
- 日志解析在状态变化后 500ms 内完成更新
- 日志解析覆盖 ≥ 90% GameState 字段，不足部分截图补充
- GameState 对比人工记录，准确率 ≥ 99%（50局样本）
- 日志监听持续运行 4 小时无内存泄漏，单条解析 ≤ 5ms

### AI 决策质量
- 全自动 100 局平均排名 ≤ 4.5，前四率 ≥ 50%
- 天梯分数稳定在 6000-7000（连续50局统计）
- 阵容选择覆盖 ≥ 5 种不同体系
- 升本节奏正确率 ≥ 80%（人工审核20局）
- 差1三连时购买率 ≥ 90%
- 金币不足时零非法购买/升级指令
- 手牌/场面已满时零非法购买指令

### 操作执行
- 单次操作成功率 ≥ 98%（200次样本）
- 操作间隔 200-600ms（随机），100%满足下限
- 操作队列积压 > 5 时自动丢弃低优先级操作
- 0% 回合因超时未完成核心操作（100局样本）

### 用户控制
- 快捷键触发后 ≤ 200ms 暂停操作
- 手动接管期间零误触
- 恢复后基于最新 GameState 决策

### 游戏阶段识别
- 识别准确率 ≥ 99%（100局样本）
- 阶段切换延迟 ≤ 1 秒
- UNKNOWN 状态占比 ≤ 0.5%

### 技术原型验证门禁
- 日志解析覆盖率 ≥ 85% GameState 字段
- 截图识别补充字段准确率 ≥ 90%
- 连续50次买卖操作成功率 ≥ 95%
- 自计时与实际回合偏差 ≤ 3 秒

## 约束条件

| 约束项 | 要求 |
|--------|------|
| 游戏语言 | 仅简体中文客户端 |
| 操作系统 | Windows 10/11 64位 |
| 最低硬件 | CPU 4核、内存 8GB、磁盘 I/O 不被阻塞 |
| 游戏分辨率 | 截图识别方案默认 1920x1080，需配置 |
| AI 水平上限 | 6000-7000分，不追求最高段位 |
| 游戏模式 | 仅酒馆战棋，不支持标准/狂野/竞技场/乱斗 |

## 风险声明

| 风险 | 等级 | 缓解措施 |
|------|------|---------|
| **封号风险** | 高 | 操作随机化模拟人类；不修改内存/注入代码；UI 中明确告知用户风险 |
| 日志格式变更 | 中 | 解析模块化便于适配；失败时降级截图识别 |
| 截图识别准确率 | 中 | 仅作备用补充，非主数据源 |
| 反作弊检测 | 中 | 鼠标贝塞尔曲线+间隔抖动 |
| 新版本新卡池 | 低 | 随从数据库可配置更新 |
